conceptlab

A ground-truth benchmark for validating transformer interpretability methods — methodology, how to read the results, and what they do and don't tell us.
📊 Live reports Code on GitHub

The problem

Interpretability research has a validation gap. When a sparse autoencoder reports that a language model represents "days of the week" as specific features, there is no way to check whether those are the features the model actually uses — the ground truth doesn't exist. Methods are typically judged by proxy signals (reconstruction loss, human ratings of feature interpretability, downstream steering effects), all of which can be satisfied by a method that is confidently wrong about the underlying computation.

conceptlab inverts the problem: instead of asking whether a method's answer about a real model is right, we build tiny worlds where the right answer is known by construction, and check whether the method finds it. A method that fails here has no business being trusted on a frontier model. (The converse doesn't hold — more on that at the end.)

Methodology

1 · Ground truth (known) Concepts are planted as anchor directions in an embedding space RD. Samples are built as x = Σ zᵢ·anchorᵢ + noise — overlapping Gaussian blobs. The label is a known boolean composite, e.g. y = (z₀ AND z₁) XOR z₂.
→
2 · Model (trained) A toy MLP or 1-layer transformer is trained to predict the label from the embeddings, and must reach high accuracy before any method runs — otherwise there is nothing coherent to interpret.
→
3 · Methods (under test) Each method sees only (model, data). It must (a) rediscover the planted concepts, and (b) rank their importance for the label. Its answers are scored against layer 1.

The decisive advantage of synthetic data is causal ground-truth importance. To learn how much concept z₂ really matters, we don't need any attribution method: we toggle z₂ in the generator, rebuild the embedding with the same noise, run the model, and measure how much its output moves. Attribution methods are then graded against this intervention — the quantity they are supposed to estimate, computed exactly.

Concept geometry is configurable, because geometry is where interesting failures live: independent orthogonal blobs (the easy case), correlated anchors at a chosen angle, sparse superposition (more concepts than dimensions, with rare concept firing — the regime the superposition hypothesis actually describes), and circular manifolds — an ordinal concept traced around a ring, mimicking the circular weekday/month geometry found in real LLMs. Blob noise is set high enough that the Gaussians genuinely overlap: no score in the current reports is saturated at 1.0, so differences between methods are always visible.

The metrics

Recovery (matched cosine)
Discovered directions are assigned to true anchors by Hungarian matching on cosine similarity; we report the mean matched cosine. 1.0 = the method's dictionary contains the planted concepts exactly.
Coverage & redundancy → capture regime
Coverage: fraction of true concepts that at least one discovered feature aligns with (cos > 0.7). Redundancy: how many features align with each covered concept. Together they classify the capture regime: compact (~1 feature per concept — the ideal), dilution (several partial features share a concept), shattering (features approach one-per-manifold-point), partial (concepts simply missed).
Ring coverage
For circular concepts: the fraction of the ring's angular bins that some discovered feature points into. A linear method finds the 2 plane axes (coverage 0.25); a diluting SAE tiles arcs all the way around.
Importance faithfulness (Spearman ρ)
Rank correlation between the method's per-concept importance and the causal ground truth from generator interventions. This is the metric that separates methods most sharply.
Interaction detection
Does the method assign non-trivial importance to every concept in an interacting group (e.g. both sides of an AND inside a XOR)? Linear methods characteristically fail this.
Sufficiency
Train a small probe from the method's discovered-concept coefficients to the label. High accuracy = the discovered set carries everything the label needs, even if individual features are imperfect.

The methods, honestly

Linear probe skyline (supervised)

Ridge-regresses the true concept coefficients from the embeddings — it is given the answer and asked to express it linearly. It is not a real method; it is the ceiling that defines what "good" means for everything below it, and a canary: if the skyline can't recover concepts, the harness (not a method) is broken.

  • Calibrates every other score
  • Needs ground-truth labels, so it cannot exist outside a benchmark

PCA

Finds orthogonal directions of maximal variance.

  • Fast, deterministic, no hyperparameters
  • Variance is the wrong objective: when concepts fire with similar frequency and scale, the covariance is nearly isotropic across the concept subspace, and PCA's axes are an arbitrary rotation of the truth. It recovers the subspace, not the basis
  • Hard-limited to D orthogonal directions — structurally cannot handle superposition

ICA

Finds directions whose coefficients are statistically independent and non-Gaussian.

  • Near-perfect here — binary concept activations are exactly the independent, non-Gaussian sources ICA assumes
  • That is also the caveat: our generator is ICA's generative model. Its dominance measures assumption-match, not general superiority — real-model features are correlated and hierarchical, which breaks the independence assumption ICA needs

Sparse autoencoders (ReLU+L1, TopK)

Learn an overcomplete dictionary that reconstructs the data under a sparsity penalty — the workhorse of current LLM interpretability, which is why they are the benchmark's main subject.

  • The only unsupervised method here that scales past D features — sparsity, not orthogonality, is its constraint, so superposition is its home turf (it beats PCA there)
  • Failure modes are informative and reproduce published LLM phenomenology (splitting, dilution) on demand
  • Feature splitting: with a large dictionary, one true concept fragments into several near-duplicate atoms
  • Manifold blindness: atoms are linear directions, so a circular concept is covered by arcs — no single feature is the ring (the dilution regime)
  • Sensitive to L1/K/dictionary-size hyperparameters, and dead atoms must be filtered before interpreting anything

Linear label probe (importance baseline)

Regresses the label on concept coefficients; importance = |weight|.

  • Cheap, and honest about what correlation alone can see
  • Blind to interactions by construction: under XOR each concept's marginal correlation with the label is exactly zero, so its weights collapse to noise — this is the benchmark's sharpest negative result

Integrated gradients

Integrates the model's gradient along a path that scales a concept's component from absent to present; attribution = accumulated gradient × component.

  • Sees interactions (the path passes through regions where the AND partner is active), and satisfies completeness by construction
  • Requires gradients and a meaningful baseline point; path choice is a real degree of freedom — this implementation is deliberately the scaffold for prototyping "integrated conceptual sensitivity" variants against exact ground truth

Ablation & activation patching

Project out (ablation) or replace with its dataset mean (patching) the component along a concept direction, and measure the change in the model's output.

  • Causal rather than correlational — the closest in-model analogue of the generator's own ground-truth intervention, and reliably the top faithfulness scorers
  • Interventions along non-orthogonal directions disturb neighbouring concepts (an issue precisely in superposition, where it matters most)
  • Ablating to zero can push activations off-distribution; mean-patching is the gentler variant, and their disagreement is itself a diagnostic

Why the results look the way they do

ICA ≈ 1.0 recovery, everywhere. Not because ICA is the best interpretability method, but because the toy world satisfies its assumptions exactly. Treat it as a second skyline: the best any assumption-matched linear method could do. The moment concepts become correlated (a config switch away), this advantage should evaporate — that's a planned follow-up experiment.

PCA lags even in the easy setting. All six concepts fire with p=0.5 at equal scale, so no direction in the concept subspace has privileged variance. PCA finds the right 6-dimensional subspace but reports an arbitrary orthogonal rotation of it — respectable sufficiency (the subspace is intact), mediocre recovery (the basis is wrong). This is worth internalising: a method can be useful for downstream prediction while being wrong about every individual concept.

In sparse superposition, the TopK SAE is the only unsupervised method that can win. The superposition experiment plants 32 concepts in 24 dimensions with sparse background firing (~5 concepts active per sample). PCA and ICA are structurally capped at 24 components, so at least 8 concepts are unrecoverable for them no matter how good they are — ICA still nails the directions it can express (high cosine) but its coverage tops out. The overcomplete TopK SAE has no such cap and recovers nearly the full set. This is the cleanest demonstration of the actual argument for SAEs: not that sparse coding is generically better, but that overcompleteness is the only way past the dimension limit.

ReLU+L1 lags TopK on recovery at every penalty setting we tried. The L1 penalty taxes activation magnitude, which both shrinks coefficients and rewards atoms that mix several true concepts (one mixed atom fires less in total than the two clean atoms it replaces). TopK's hard constraint taxes nothing, so its atoms stay clean. This reproduces, in miniature, the motivation for the TopK/BatchTopK family in the SAE literature — and it means SAE benchmark results are substantially a property of the sparsity mechanism, not just of "SAEs" as a class.

The label probe's collapse on XOR (ρ ≈ 0.03) is the benchmark's sharpest result. Under y = (z₀ AND z₁) XOR z₂, flipping any single concept flips the label for some inputs — every concept is causally important — yet each concept's marginal correlation with the label is zero by symmetry. Correlational importance is structurally, not accidentally, blind here, while ablation/patching/IG (which go through the model) score 0.75–0.9. Any importance method proposed for real models should be required to pass this gate first.

SAE dilution on the ring is geometry, not a bug. A ring in a 2-plane cannot be written as a sparse non-negative combination of a few fixed directions; the reconstruction- optimal solution under sparsity pressure is a fan of unit atoms tiling the circle, each active for an arc. That is what the report's embedding-space panel shows, what ring-coverage ≫ 0.25 quantifies, and what was observed in real LLMs (weekday circles in Llama). The toy result and the frontier-model result agreeing is the strongest evidence the benchmark measures something real.

ReLU+L1 and TopK dilute differently — the sparsity mechanism matters. In the circle experiment the two SAE variants are configured identically except for how sparsity is enforced, and they land in different regimes: the L1-penalised SAE stays roughly compact (a few ±axis-aligned atoms, ring coverage ~0.25–0.4) because L1 taxes total activation mass and lets two plane axes share the reconstruction, while TopK (k=2) — which forces exactly k atoms per sample and taxes nothing else — recruits a fan of dedicated arc atoms (ring coverage ~0.75, redundancy > 2.5: dilution). The practical reading: which capture regime you observe in a real model is partly an artifact of your SAE's sparsity mechanism, not purely a property of the model being studied.

Why the causal methods agree with each other so closely. On these small models with near-orthogonal planted directions, ablation, patching, and IG estimate the same counterfactual and converge. Their divergence grows exactly where interpretability gets hard — non-orthogonal anchors, stronger superposition — which makes disagreement between causal methods a useful instability signal at scale, not just noise.

What transfers to real models — and what doesn't

Passing conceptlab is necessary, not sufficient. The benchmark is a filter for methods that fail even when everything is easy — linear planting, known concept count, IID samples, a model that provably learned the label. A method that aces it can still fail on a frontier model; a method that flunks it should not be trusted anywhere.

What transfers well: the failure modes. They are structural, so they scale. Correlational importance will stay blind to interactions in a 70B model for the same mathematical reason it is blind here. SAE atoms will stay linear, so curved concept manifolds will keep diluting. Variance will keep failing to align with semantics for PCA-style methods. The benchmark's job is to make these failure modes cheap to demonstrate and quantify.

What transfers poorly: the absolute numbers and the rankings among survivors. Real models differ in ways the toy world deliberately omits:

  • Nonlinear feature embedding. We plant concepts as fixed directions in the input space. Real features emerge across depth, can be nonlinearly composed, and may only be linear locally. Recovery scores here are upper bounds.
  • Correlated, hierarchical concepts. Real concepts co-occur ("Paris" and "France") and nest. ICA's win depends on independence; expect it to degrade first.
  • Unknown concept count. We can set the SAE dictionary relative to the true number of concepts. Real work must guess, and the splitting/dilution trade-off is governed by that guess.
  • Distribution shift and off-manifold interventions. Ablation on a real model can push activations far off-distribution, where model behaviour is uninformative; our blobs are forgiving about this.
  • Benchmark Goodharting. A method tuned to this generator family (sparse linear mixtures + rings) inherits its assumptions. The defence is config diversity: correlated anchors, hierarchies, and label families the method's authors didn't see.

The recommended use pattern is therefore regression testing, not leaderboard chasing: when developing a new method (say, a variant of integrated conceptual sensitivity), first show it clears the gates existing methods clear, then show it survives a toy pathology they don't — and only then spend the compute to look for that pathology in a real model.

Reading a report

Each run report has four panels. Suggested reading order: first the meta line (val accuracy — if the model didn't learn the label, stop); then the scorecard (recovery and regime tell you whether concepts were found and in what shape; faithfulness ρ tells you whether importance can be trusted); then the per-concept importance chart (compare each method's bars against the gray ground-truth bars — this is where the label probe's XOR failure is visible to the naked eye); and finally the embedding-space projection (green = planted anchors, yellow = what the method found — splitting and dilution are directly visible as yellow fans).
